UAC (User Account Control) adalah fitur keamanan di Windows yang:

  • Mencegah program berjalan dengan hak administrator tanpa izin.
  • Memastikan bahwa pengguna sadar dan menyetujui sebelum menjalankan tindakan sistem-level.

üßç‚Äç‚ôÇÔ∏è Masalahnya: Sebagian Besar Pengguna Rumah Login Sebagai Administrator

  • Pengguna rumahan biasanya login sebagai akun administrator, karena:
    • Default saat menginstal Windows
    • Mudah untuk menginstal program, ubah pengaturan, dsb.

‚ùó Risiko:

  • Malware yang berjalan di akun admin bisa langsung mengubah sistem.
  • Tanpa perlindungan, semua tindakan berjalan dengan hak penuh.

üõ°Ô∏è Solusi: UAC

üîß Cara Kerja UAC:

  1. Kamu login sebagai administrator.
  1. Namun, Windows hanya memberikan hak terbatas secara default.
  1. Saat program ingin melakukan hal sistem-level (contoh: instalasi), UAC akan memunculkan prompt izin.

üîë Contoh:

  • Saat menjalankan installer, muncul jendela seperti ini:
Do you want to allow this app to make changes to your device?
  • Kamu harus:
    • Klik ‚ÄúYes‚Äù jika kamu adalah admin
    • Atau masukkan password admin jika login sebagai user biasa

üë®‚Äçüîß Catatan:

  • Akun built-in Administrator (yang tersembunyi secara default) tidak dipengaruhi oleh UAC ‚Äî semua tindakan berjalan dengan hak penuh.
  • Makanya, tidak disarankan menggunakan akun Administrator bawaan untuk sehari-hari.

üß† Contoh Kasus dalam Praktik:

SituasiApa yang Terjadi?

Login sebagai Administrator bawaanTidak ada UAC, program langsung dijalankan dengan hak penuh
Tidak klik apapun di prompt UACInstaller tidak akan berjalan (tidak diberi izin)

‚ö†Ô∏è Mengapa Penting?

Tanpa UAC:

  • Malware dapat langsung melakukan:
    • Mengubah registry
    • Menulis ke direktori sistem (C:\Windows, C:\Program Files)
    • Memasang driver atau backdoor

Dengan UAC:

  • Kamu tetap bisa menjadi administrator, tapi:
    • Diberi "rem pengaman"
    • Kamu harus mengonfirmasi sebelum sistem diubah
