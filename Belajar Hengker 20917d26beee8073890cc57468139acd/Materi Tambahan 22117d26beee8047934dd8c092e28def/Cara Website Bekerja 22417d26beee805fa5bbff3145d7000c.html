Cara Website Bekerja

üåê Bagaimana Website Bekerja

1. Proses Dasar Saat Mengakses Website:

Ketika kamu mengunjungi sebuah website (misalnya google.com), browser seperti Chrome atau Safari akan:

  • Mengirim permintaan (request) ke komputer lain bernama web server.
  • Server ini akan membalas (response) dengan data (biasanya berupa HTML, CSS, JavaScript).
  • Browser akan menampilkan halaman berdasarkan data yang diterima.

üß† Web server = komputer yang menyimpan dan mengirimkan halaman web.

üß± Komponen Website

‚úÖ Front-End (Client-Side)

Bagian yang dilihat dan digunakan oleh pengguna di browser:

  • Dibuat dengan:
    • HTML ‚Üí struktur
    • CSS ‚Üí tampilan / desain
    • JavaScript ‚Üí interaksi / fungsionalitas

‚úÖ Back-End (Server-Side)

Bagian tersembunyi yang memproses data dan mengelola permintaan dari pengguna (misalnya mengambil data dari database, otentikasi login, dsb).

üß© Dasar-Dasar HTML

HTML (HyperText Markup Language) adalah bahasa untuk membuat struktur halaman web.

Contoh struktur sederhana:

<!DOCTYPE html>
<html>
	<head>
		<title>judul_halaman</title>
	</head>
	<body>
		<h1>Selamat Datang</h1>
		<p>Ini adalah paragraf</p>
	</body>
</html>

Penjelasan:

  • <!DOCTYPE html> ‚Üí memberi tahu browser bahwa ini adalah dokumen HTML5.
  • <html> ‚Üí elemen utama, semua isi HTML ada di dalamnya.
  • <head> ‚Üí berisi informasi tersembunyi seperti judul halaman.
  • <body> ‚Üí isi utama yang ditampilkan ke pengguna.
  • <h1> ‚Üí heading (judul besar).
  • <p> ‚Üí paragraf.

HTML juga memiliki:

  • class ‚Üí untuk memberi nama kelompok, bisa digunakan oleh CSS/JS.
  • id ‚Üí unik, hanya digunakan untuk satu elemen, biasanya dipakai oleh JavaScript.

Contoh:

<p class="teks-merah" id="paragraf1">Halo Dunia</p>

‚ú® JavaScript: Membuat Halaman Interaktif

JavaScript (JS) digunakan agar halaman web bisa:

  • Merespons klik tombol
  • Menampilkan animasi
  • Mengubah teks/gambar tanpa me-refresh

Contoh JavaScript:

<script>
  document.getElementById("demo").innerHTML = "Hack the Planet";
</script>

Penjelasan:

  • getElementById("demo") mencari elemen HTML yang punya id="demo"
  • innerHTML = "..." mengubah isi elemen tersebut

Contoh tombol interaktif:

<button onclick='document.getElementById("demo").innerHTML = "Tombol Diklik"'>Klik Saya!</button>
<p id="demo">Teks ini akan berubah</p>

üîê Sensitive Data Exposure (Kebocoran Data Sensitif)

üîé Apa itu?

Sensitive Data Exposure terjadi ketika sebuah website menampilkan informasi penting/sensitif secara tidak sengaja di halaman depan (frontend), seperti:

  • Username & password
  • Link ke halaman rahasia
  • API key, token, atau konfigurasi
  • Catatan internal dalam komentar HTML/JavaScript

‚ö†Ô∏è Contoh:

Seorang developer lupa menghapus komentar:

<!-- TODO: Hapus akun admin sementara: admin123 / pass123 -->

Kalau seseorang klik kanan ‚Üí "Lihat Sumber Halaman", data ini akan terlihat.

üéØ Bahayanya:

Penyerang bisa:

  • Menggunakan informasi itu untuk login
  • Akses bagian tersembunyi dari website
  • Melanjutkan serangan ke server backend atau database

‚úÖ Cara Menghindari:

  • Jangan pernah menyimpan data sensitif di HTML atau JavaScript
  • Hapus komentar internal sebelum situs dirilis
  • Gunakan tools scanning otomatis (seperti OWASP ZAP)

üß® HTML Injection

üîé Apa itu?

HTML Injection adalah kerentanan di mana input dari pengguna dimasukkan ke halaman tanpa disaring.

Hasilnya, penyerang bisa menyisipkan kode HTML atau JavaScript yang aktif di halaman.

‚ö†Ô∏è Contoh:

Misalnya ada form:

Nama: <input name="nama">

Dan halaman menampilkan:

Halo, <?php echo $_GET['nama']; ?>

Jika kamu input:

<h1>HACKED</h1>

Maka halaman akan menampilkan judul besar ‚ÄúHACKED‚Äù.

Jika kamu input JavaScript:

<script>alert('XSS');</script>

Browser akan menjalankannya!

üéØ Bahayanya:

  • Menipu pengguna (phishing)
  • Mengubah tampilan halaman
  • Melancarkan serangan XSS (Cross-Site Scripting)

‚úÖ Cara Mencegah:

  • Sanitasi input ‚Üí filter atau hapus tag HTML/JavaScript
  • Jangan pernah langsung menampilkan input pengguna tanpa pengecekan
  • Gunakan textContent (bukan innerHTML) dalam JavaScript
