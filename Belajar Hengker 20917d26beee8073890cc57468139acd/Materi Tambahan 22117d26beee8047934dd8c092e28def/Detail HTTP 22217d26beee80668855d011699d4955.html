Detail HTTP

HTTP (HyperText Transfer Protocol) adalah protokol komunikasi yang digunakan saat kita mengakses halaman web. Protokol ini dikembangkan oleh Tim Berners-Lee dan timnya antara tahun 1989‚Äì1991.

HTTP digunakan untuk mengirim dan menerima data seperti:

  • HTML (struktur halaman web)
  • Gambar
  • Video
  • Script, dan lainnya

HTTPS (HTTP Secure) adalah versi aman dari HTTP. Bedanya:

  • Data yang dikirim dienkripsi, sehingga tidak bisa disadap oleh pihak ketiga.
  • HTTPS juga memastikan bahwa kamu benar-benar terhubung ke server yang sah (bukan server palsu).
  • Menggunakan protokol tambahan: TLS (Transport Layer Security)

Tanda bahwa situs menggunakan HTTPS:

  • Ada ikon gembok üîí di address bar browser
  • URL dimulai dengan https://

URL (Uniform Resource Locator) adalah alamat lengkap dari suatu sumber daya (resource) di internet. Saat kamu mengakses situs apa pun, kamu sedang menggunakan URL.

Contoh URL lengkap:

http://user:password@tryhackme.com:80/view-room?id=1#task3

Ketika kamu mengunjungi sebuah situs web, browser kamu akan mengirimkan HTTP Request ke server. Ini seperti kamu berkata:

"Hai server, tolong kirimkan halaman ini kepadaku."

Contoh paling sederhana dari HTTP Request:

GET / HTTP/1.1

Namun, untuk komunikasi yang lebih kaya, kamu juga perlu mengirim informasi tambahan melalui header. Header berisi metadata, informasi tambahan tentang permintaanmu, seperti browser yang digunakan, asal permintaan, dan sebagainya.

Headers adalah data tambahan yang dikirim bersama permintaan (request) atau balasan (response) dalam komunikasi HTTP. Meskipun secara teknis tidak wajib, tapi tanpa headers, website tidak akan bisa ditampilkan dengan baik.

üîº Request Headers (Dari Klien ‚Üí Server)

Header ini dikirim oleh browser atau aplikasi saat meminta data ke server:

HeaderFungsi
HostMenentukan nama domain yang diminta. Wajib jika satu server punya banyak domain (hosting bersama).
User-AgentMenjelaskan jenis dan versi browser, agar server bisa menyesuaikan konten.
Content-LengthMemberi tahu server seberapa besar data yang dikirim (misalnya saat login atau upload).
Accept-EncodingMenyatakan jenis kompresi data yang bisa diterima oleh browser (misal: gzip, br).
CookieMengirimkan data sesi atau identitas pengguna ke server (lihat bagian "Cookie").

üîΩ Response Headers (Dari Server ‚Üí Klien)

Header ini dikirim oleh server sebagai respon terhadap permintaan HTTP:

HeaderFungsi
Set-CookieMengirim data ke browser untuk disimpan sebagai cookie, yang akan dikirim balik ke server di permintaan berikutnya.
Cache-ControlMenentukan berapa lama browser boleh menyimpan konten ini sebelum minta ulang.
Content-TypeMenjelaskan tipe konten: HTML, CSS, JSON, gambar, video, dll.
Content-EncodingMenjelaskan metode kompresi yang dipakai pada konten.

Cookie adalah data kecil yang disimpan oleh browser. Cookie biasanya berisi:

  • ID sesi login
  • Pengaturan personal (bahasa, tema)
  • Status login (kamu sudah login atau belum)
  • Token otentikasi

üîÅ Cara kerja cookie:

  1. Server mengirimkan header:

    Set-Cookie: session=abc123;

  1. Browser menyimpan cookie.
  1. Setiap permintaan berikutnya akan otomatis menyertakan:

    Cookie: session=abc123

  1. Server bisa mengenali kamu sebagai pengguna yang sama.
HTTP bersifat stateless, artinya setiap permintaan dianggap baru, tanpa riwayat sebelumnya. Cookie dipakai agar server ingat siapa kamu.
