Apa Itu DNS

Bayangkan kamu ingin mengunjungi situs web seperti tryhackme.com, tapi tanpa DNS, kamu harus mengetik IP seperti ini:

104.26.10.229

Susah diingat, kan?

Nah, DNS (Domain Name System) adalah sistem yang berfungsi mengubah nama domain (seperti tryhackme.com) menjadi IP address (seperti 104.26.10.229) agar kamu tidak perlu mengingat angka-angka rumit.

üè† Analogi DNS

Seperti buku telepon untuk internet.

Kamu ingin menelepon ‚ÄúPizza Hut‚Äù, tapi kamu tidak tahu nomor teleponnya. Kamu buka buku telepon, cari "Pizza Hut", lalu dapatkan nomornya.

Begitu juga dengan DNS: kamu ketik tryhackme.com, DNS mencari nomor IP-nya dan menghubungkanmu.

  • ccTLD (Country Code): .id, .my, .jp, .uk, .us

üîß Jenis-jenis Record DNS

DNS tidak cuma dipakai untuk situs web. Ada banyak jenis record DNS, dan ini adalah yang paling sering kamu temui:

üß† Langkah-langkah Resolusi DNS

üîç 1. Cek Cache Lokal

Komputer kamu akan mengecek cache lokal dulu:

  • Pernahkah kamu membuka domain ini sebelumnya?
  • Jika pernah, IP-nya masih disimpan sementara (cached), dan proses berhenti di sini ‚Üí kamu langsung terhubung!

Jika tidak ada di cache, proses lanjut ke langkah berikutnya.

üåê 2. Kirim ke Recursive DNS Server

Permintaan dikirim ke Recursive DNS Server (biasanya milik ISP, atau bisa kamu ganti ke Google DNS, Cloudflare, OpenDNS, dll).

  • DNS ini juga punya cache sendiri.
  • Jika domain sudah pernah dicari sebelumnya, jawabannya langsung dikembalikan ke kamu ‚Üí hemat waktu dan bandwidth.

Jika tidak ada juga, proses lanjut ke perjalanan utama DNS.

üåç 3. Akses Root DNS Server

Root server adalah tulang punggung DNS di internet.

  • Fungsinya: mengarah ke server TLD (Top-Level Domain) yang tepat.
  • Contoh: untuk www.tryhackme.com, root server akan bilang:
    ‚ÄúIni domain .com, silakan tanya ke server TLD .com!‚Äù

üè¢ 4. Arahkan ke TLD Server

Server TLD .com menyimpan informasi ke mana kita harus bertanya untuk tahu alamat IP domain yang dimaksud.

  • Misalnya: TLD .com akan memberi tahu bahwa untuk tryhackme.com, kamu harus tanya ke:
    kip.ns.cloudflare.com dan uma.ns.cloudflare.com (Nameserver)

üì° 5. Tanya ke Authoritative DNS Server

Ini adalah server yang memiliki data DNS asli dari domain tersebut.

  • Misalnya, owner dari tryhackme.com menyimpan semua record DNS (A record, CNAME, MX, dll) di server ini.
  • Server ini akan menjawab dengan informasi yang dibutuhkan, misalnya:
    A record ‚Üí 104.26.10.229

üîÅ 6. Kembalikan Jawaban ke Recursive DNS

  • Jawaban dari Authoritative DNS Server dikembalikan ke Recursive DNS
  • Recursive DNS menyimpan hasil ini di cache ‚Üí jadi jika ada permintaan serupa di masa depan, tidak perlu ulang proses panjang

üíª 7. Kembalikan ke Komputermu

  • Recursive DNS mengirim IP address ke komputermu
  • Sekarang browser tahu harus ke mana: langsung terhubung ke server tujuan (misalnya 104.26.10.229)

üïí TTL (Time To Live)

Semua record DNS memiliki TTL (Time To Live):

  • TTL menentukan berapa lama hasil itu disimpan dalam cache
  • Misalnya: TTL = 3600 ‚Üí simpan 1 jam
  • Setelah itu, perlu permintaan ulang agar dapat data terbaru

üîç 1. A Record

nslookup --type=A website.thm

Hasil:

Server: 127.0.0.53
Address: 127.0.0.53#53

Non-authoritative answer:
Name: website.thm
Address: 10.10.10.10

üîé Makna:

  • Server: 127.0.0.53 ‚Äì adalah resolver lokal (biasanya DNS cache system di Linux, seperti Systemd-resolved).
  • Non-authoritative answer: artinya jawaban ini berasal dari cache resolver, bukan langsung dari authoritative DNS server.
  • Nama domain website.thm diarahkan ke alamat IP versi IPv4 10.10.10.10.

üîÅ 2. CNAME Record

Hasil:

Server: 127.0.0.53
Address: 127.0.0.53#53

Non-authoritative answer:

shop.website.thm canonical name = shops.myshopify.com

üîé Makna:

  • Server: 127.0.0.53 ‚Äì adalah resolver lokal (biasanya DNS cache system di Linux, seperti Systemd-resolved).
  • Non-authoritative answer: artinya jawaban ini berasal dari cache resolver, bukan langsung dari authoritative DNS server.
  • Subdomain shop.website.thm adalah alias (CNAME) dari domain shops.myshopify.com.
    ‚úÖ Browser akan lanjut melakukan query ke
    shops.myshopify.com untuk mendapatkan IP aslinya.

    ‚úÖ Google menangani email domain ini. Jika kamu kirim email ke
