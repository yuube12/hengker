Yang Terjadi Saat Mengakses Website

  1. DNS: Komputer kamu mencari tahu alamat IP dari nama domain yang kamu masukkan (misalnya google.com).
  1. HTTP: Komputer kamu mengirim permintaan (request) menggunakan protokol HTTP ke web server tujuan.
  1. Server Web: Server merespons dengan mengirim file HTML, CSS, JavaScript, gambar, dll.
  1. Browser: Browser kamu membaca file tersebut dan menampilkan halaman web di layar.

✅ 1. Load Balancer

Digunakan untuk membagi beban pengunjung ke beberapa server.

Fungsi utama:

  • Menangani lalu lintas tinggi agar tidak overload.
  • Memberikan cadangan otomatis jika salah satu server mati.

Cara kerja:

  • Permintaan web dikirim dulu ke load balancer.
  • Load balancer memilih server paling cocok, berdasarkan:
    • Round-robin: mengirim bergiliran ke semua server.
    • Weighted: memilih server dengan beban kerja paling ringan.
  • Melakukan health check ke tiap server secara rutin.

✅ 2. CDN (Content Delivery Network)

Digunakan untuk mengirim file statis (gambar, CSS, JavaScript, video) dari server terdekat.

Keuntungan:

  • Mempercepat loading website dari seluruh dunia.
  • Mengurangi beban server utama.
  • File disalin ke ribuan server di seluruh dunia.

Contoh CDN: Cloudflare, Akamai, Amazon CloudFront

✅ 3. Database

Digunakan untuk menyimpan dan mengambil data pengguna.

Contoh penggunaan:

  • Menyimpan akun/login
  • Menyimpan komentar atau postingan
  • Menyimpan produk di toko online

Jenis database umum:

  • MySQL, PostgreSQL → relasional
  • MongoDB → NoSQL
  • MSSQL → buatan Microsoft

✅ 4. WAF (Web Application Firewall)

Melindungi website dari serangan siber.

Fungsi utama:

  • Memeriksa permintaan dari pengguna: apakah itu dari browser asli atau dari bot.
  • Menangkal serangan seperti:
    • SQL injection
    • XSS (Cross-site scripting)
    • Brute force
    • DDoS ringan
  • Rate limiting: membatasi jumlah request dari satu IP dalam satu detik.

Jika ada permintaan mencurigakan → langsung diblokir sebelum mencapai server web.

✅ Definisi:

Web server adalah perangkat lunak (software) yang:

  1. Mendengarkan koneksi dari klien (seperti browser)
  1. Menggunakan protokol HTTP untuk mengirimkan konten web (HTML, gambar, CSS, dll)

Jadi, saat kamu membuka website seperti http://example.com, browser kamu menghubungi web server, dan server akan mengirimkan file yang diminta.

📁 Root Directory (Folder Akar)

Web server mengirim file dari folder khusus yang disebut root directory — tempat semua file web disimpan.

Web ServerLokasi Default (Root Directory)
Apache & Nginx/var/www/html (di Linux)
IISC:\inetpub\wwwroot (di Windows)

🧪 Contoh:

Jika kamu mengakses:

http://www.example.com/picture.jpg

Web server akan mengirimkan file dari:

/var/www/html/picture.jpg (jika pakai Linux)

Artinya file picture.jpg harus benar-benar ada di folder tersebut, kalau tidak maka akan muncul error 404 (file not found).

✅ Pengertian:

Virtual Host adalah fitur pada web server yang memungkinkan satu server fisik melayani banyak website dengan domain berbeda.

Dengan Virtual Host, kamu bisa meng-host:

  • www.satu.com
  • www.dua.com
  • www.apapun.com

    ...semuanya di satu server saja.

🧠 Bagaimana Cara Kerjanya?

  1. Saat browser mengakses website (www.satu.com), ia mengirim permintaan HTTP yang berisi hostname di header.
  1. Web server akan membaca hostname itu.
  1. Web server mencocokkannya dengan konfigurasi virtual host.
  1. Jika ada cocokan, maka akan menampilkan halaman dari folder yang sesuai.
  1. Jika tidak ada cocokan, maka akan menampilkan website default.

📁 Contoh:

DomainRoot Folder
satu.com/var/www/satu
dua.com/var/www/dua

Jadi, jika kamu mengakses:

http://satu.com

Server akan memberikan isi dari:

/var/www/satu

⚙️ Konfigurasi Virtual Host di Apache (Contoh)

Contoh file konfigurasi Apache (/etc/apache2/sites-available/satu.com.conf):

<VirtualHost *:80>
  ServerName satu.com
  DocumentRoot /var/www/website_satu
</VirtualHost>

Dan file lainnya untuk dua.com:

<VirtualHost *:80>
  ServerName dua.com
  DocumentRoot /var/www/website_dua
</VirtualHost>

Setelah itu, kamu aktifkan konfigurasi dan restart Apache:

sudo a2ensite satu.com.conf
sudo a2ensite dua.com.conf
sudo systemctl restart apache2

🚫 Jika Tidak Ada Kecocokan?

Kalau kamu akses domain yang belum dikonfigurasi (misalnya tiga.com), web server akan menampilkan website default.

🧱 1. Konten Statis (Static Content)

✅ Apa itu?

Konten yang selalu sama, tidak berubah, dan langsung dikirim dari server tanpa diproses dulu.

📂 Contoh:

  • Gambar (.jpg, .png)
  • File CSS (style.css)
  • File JavaScript (script.js)
  • HTML sederhana yang isinya tidak pernah berubah (index.html)

⚙️ Cara kerja:

Server hanya mengirim file apa adanya. Tidak peduli siapa yang mengakses, hasilnya selalu sama.

🔄 2. Konten Dinamis (Dynamic Content)

✅ Apa itu?

Konten yang berubah-ubah tergantung permintaan. Biasanya diproses oleh bahasa pemrograman backend.

📂 Contoh:

  • Homepage blog yang menampilkan artikel terbaru
  • Halaman profil pengguna yang berbeda tergantung siapa yang login
  • Halaman pencarian yang hasilnya berubah tergantung kata yang diketik

⚙️ Cara kerja:

  1. Pengguna mengirim permintaan (contoh: ?name=adam)
  1. Server menjalankan kode backend (misalnya PHP, Python, NodeJS)
  1. Hasil pemrosesan dikirim ke browser sebagai HTML

🔧 Contoh Kasus: Dynamic dengan PHP

Permintaan ke URL:

http://example.com/index.php?name=adam

Isi file index.php:

<html><body>Hello <?php echo $_GET["name"]; ?></body></html>

Hasil yang dilihat pengguna:

<html><body>Hello adam</body></html>

🧠 Perhatikan:

  • Browser tidak pernah melihat kode PHP-nya karena PHP diproses di backend/server.
  • Yang dikirim ke browser hanya hasil akhirnya dalam bentuk HTML.

BagianFungsiContoh Teknologi
FrontendApa yang dilihat penggunaHTML, CSS, JavaScript
BackendApa yang diproses di serverPHP, Python, NodeJS

🔒 Dampak Keamanan:

Karena konten dinamis bisa menerima input dari pengguna dan memprosesnya di server, ini bisa:

  • Membuka celah serangan seperti SQL Injection, XSS, dll
  • Menyebabkan data bocor jika server tidak memfilter input dengan benar
⚠️ Semakin dinamis sebuah web, semakin tinggi risiko keamanannya jika tidak dirancang dengan baik.

🧠 Kesimpulan:

Konten StatisKonten Dinamis
Tidak berubahBerubah tergantung input/user
Langsung dikirim dari serverDiproses dulu oleh backend
Contoh: gambar, CSS, HTMLContoh: hasil pencarian, profil
Lebih aman & cepatLebih fleksibel, tapi rawan celah

🔹 1. Browser Meminta Halaman

Kamu mengetik https://tryhackme.com dan menekan Enter.

🔹 2. Periksa Cache Lokal

Browser atau sistem operasi akan mengecek cache DNS lokal (apakah sudah pernah mengakses tryhackme.com sebelumnya).

Kalau alamat IP-nya masih tersimpan, akan langsung digunakan.

Kalau tidak ada, maka lanjut ke langkah 3.

🔹 3. Tanya ke Recursive DNS Resolver

Komputermu menghubungi DNS resolver (biasanya milik ISP atau Google: 8.8.8.8) untuk mencari tahu alamat IP dari tryhackme.com.

🔹 4. DNS Resolver Tanya ke Root Server

Jika DNS resolver tidak tahu, dia akan:

  • Bertanya ke root DNS server (misal: “.com” ada di mana?)

🔹 5. Dari Root ke Authoritative DNS Server

  • Root server akan mengarahkan ke DNS server khusus domain .com
  • Dari sana, diarahkan lagi ke authoritative DNS server milik TryHackMe
  • DNS server itu memberikan alamat IP sebenarnya dari tryhackme.com

🔹 6. Permintaan Melewati Web Application Firewall (WAF)

  • Setelah tahu IP tujuan, permintaanmu dikirim ke sana
  • Tapi sebelum sampai ke server web, permintaanmu dicek oleh WAF
  • WAF akan:
    • Mendeteksi serangan (XSS, SQL Injection, dll)
    • Memblokir traffic berbahaya
    • Membatasi spam request

🔹 7. Permintaan Melewati Load Balancer

  • Jika situsnya sibuk atau terdiri dari banyak server, permintaan kamu diarahkan ke salah satu server yang sedang tidak sibuk menggunakan load balancer
  • Load balancer memilih server terbaik (round-robin, least connection, dll)

🔹 8. Sambungan ke Web Server (Port 80/443)

  • Browser kamu melakukan koneksi ke server menggunakan:
    • Port 80 untuk HTTP biasa
    • Port 443 untuk HTTPS (aman, terenkripsi)

🔹 9. Web Server Menerima GET Request

  • Web server (Apache, Nginx, dll) menerima permintaan dari browser:
    GET / HTTP/1.1
    Host: tryhackme.com

🔹 10. Web Server Menghubungi Web Application

  • Web server meneruskan permintaan ke web application backend (misal: PHP, Node.js)
  • Aplikasi ini yang menentukan apa yang harus ditampilkan ke user

🔹 11. Web Application Berinteraksi dengan Database

  • Jika perlu data (contoh: daftar room TryHackMe), aplikasi backend akan:
    • Mengirim query ke database (misal: MySQL, MongoDB)
    • Mendapat data dan menyusunnya jadi HTML

🔹 12. HTML Dikirim ke Browser

  • Web server mengembalikan HTML hasil akhir ke browser kamu
  • Jika ada gambar, CSS, JavaScript, browser akan memintanya satu per satu

🔹 13. Browser Merender Website

  • Browser membaca HTML, CSS, dan JavaScript
  • Lalu menampilkan website secara visual di layar kamu
