Pengaturan VM

1. NAT (Network Address Translation)

üõ† Penjelasan:

  • VM bisa internetan.
  • Terisolasi dari VM lain dan dari jaringan lokal.
  • Tidak bisa di-scan dari luar (kecuali kamu buat port forwarding).

‚úÖ Digunakan Saat:

  • Install tools, update, browsing.
  • Ingin aman dari luar, dan tidak perlu komunikasi antar VM.

‚ùå Tidak sepenuhnya anonim.

  • Memang VM tidak terlihat di jaringan lokal, tapi:
    • Semua trafik tetap lewat IP asli host kamu.
    • Jadi ISP tetap bisa tahu kamu mengakses apa.
  • Kalau kamu pakai VPN di VM, itu bisa lebih aman, tapi host tetap terlihat oleh ISP.

2. NAT Network

üõ† Penjelasan:

  • Mirip NAT, tapi bisa antar VM saling terhubung dalam 1 jaringan virtual.
  • Masih bisa internetan.
  • Perlu setup NAT Network dulu via VirtualBox Preferences.

‚úÖ Digunakan Saat:

  • Simulasi jaringan lokal antar beberapa VM (cth: attacker & korban).
  • CTF, malware lab multi-host, MITM antar VM.

‚úÖ Bisa jadi cukup anonim

  • jika:
    • VM tidak langsung pakai IP dari jaringan lokal.
    • Dan kamu menggunakan VPN/Tor langsung dari dalam VM.
  • Tapi secara default masih tidak sepenuhnya anonim ke ISP.

3. Bridged Adapter

üõ† Penjelasan:

  • VM terhubung langsung ke jaringan fisik (seperti komputer biasa).
  • Dapat IP dari router, bisa diakses dari perangkat lain di jaringan yang sama.

‚úÖ Digunakan Saat:

  • Butuh VM terlihat di jaringan nyata (misal: dicoba serang dari laptop lain).
  • Ingin uji pentest dengan target yang ada di luar VM.

‚ùå Paling tidak anonim.

  • Karena VM benar-benar muncul sebagai perangkat nyata di jaringan lokal kamu.
  • Router dan semua perangkat lain akan melihat VM seperti komputer biasa.
  • IP-nya berasal dari router rumah, dan MAC address bisa dengan mudah dilacak.
  • ISP pun tahu kamu online dengan IP itu.

4. Not Attached

üõ† Penjelasan:

  • VM tidak punya koneksi jaringan sama sekali.
  • Seperti komputer tanpa kabel LAN atau Wi-Fi.

‚úÖ Digunakan Saat:

  • Latihan pentesting offline.
  • Isolasi total untuk keamanan (cth: malware analysis).
  • Tidak ingin VM mengakses atau diakses siapa pun.

‚úÖ Paling anonim secara jaringan.

  • Karena tidak ada koneksi ke mana pun.
  • Cocok untuk analisis malware offline atau latihan tanpa risiko kebocoran data.
  • Tapi tentu saja tidak bisa browsing atau koneksi luar.

Dalam sistem operasi Linux, interface jaringan seperti eth0, wlan0, dan lo adalah "pintu" bagi perangkat untuk terhubung ke jaringan lokal maupun internet.

Setiap interface punya tugas dan jalur koneksi masing-masing.

üåê 1. eth0 ‚Äî Ethernet Interface

  • Artinya: Ini adalah interface untuk koneksi kabel (LAN).
  • Contoh penggunaannya: Komputer dihubungkan ke router/modem dengan kabel LAN.
  • Sifatnya:
    • Stabil dan cepat.
    • Biasanya muncul otomatis di VM kalau pakai NAT, Bridged, atau NAT Network.
  • Alamat IP: Biasanya didapat dari DHCP router (misalnya: 192.168.x.x).
  • Di ifconfig/ip a: Ditandai dengan nama eth0 atau enp0s3 (tergantung distribusi Linux).

üåÄ 2. lo ‚Äî Loopback Interface

  • Artinya: Jalur internal untuk komunikasi dari komputer ke dirinya sendiri.
  • Alamat IP: Selalu 127.0.0.1 (alias localhost).
  • Fungsi penting:
    • Untuk testing jaringan lokal.
    • Banyak aplikasi web lokal (seperti database, web server lokal, dsb) berjalan di sini.
  • Contoh penggunaan:
    ping 127.0.0.1
curl http://localhost

‚û°Ô∏è Tanpa lo, banyak sistem internal Linux tidak akan berfungsi dengan benar.

üì° 3. wlan0 ‚Äî Wireless LAN Interface

  • Artinya: Ini adalah interface untuk koneksi WiFi (nirkabel).
  • Muncul kapan?:
    • Hanya muncul kalau sistem mendeteksi USB WiFi adapter atau modul WiFi bawaan.
    • Di VM, harus ada USB WiFi yang disambungkan langsung ke VM.
  • Alamat IP: Sama seperti eth0, biasanya didapat dari DHCP router.
  • Nama alternatif:
    • Bisa berubah tergantung sistem menjadi wlp2s0, wlx..., dsb.
  • Gunanya dalam pentesting:
    • Untuk monitoring jaringan wireless, inject paket, atau membuat rogue AP.

üîå Apa Itu Port?

Port adalah gerbang virtual yang digunakan oleh komputer untuk mengirim dan menerima data dalam jaringan. Setiap kali kita berkomunikasi melalui jaringan (seperti membuka website, kirim email, streaming, dll), komunikasi itu terjadi melalui port tertentu.

Bayangkan port seperti pintu-pintu khusus yang masing-masing punya tugas berbeda. Misalnya:

  • Pintu 80 (port 80) untuk HTTP (website biasa)
  • Pintu 443 untuk HTTPS (website aman)
  • Pintu 22 untuk SSH (remote ke server)

Komputer bisa menggunakan banyak port dalam satu waktu, dan masing-masing punya fungsi yang berbeda berdasarkan protokol yang digunakan.

üì° Mengapa Port Penting?

  • Setiap aplikasi di jaringan memakai port tertentu.
  • Saat kamu menjalankan nmap, kamu bisa melihat port mana yang terbuka (open) pada perangkat target.
  • Port yang terbuka bisa menjadi pintu masuk serangan jika tidak dijaga dengan baik.
